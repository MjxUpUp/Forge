# PRD 模板

## 变量列表

**必填**：
- `feature_name`：功能名称
- `goal_one_liner`：一句话目标（这个功能解决什么问题）
- `target_user`：目标用户（谁用，角色/画像）

**选填**（不提供则生成草稿标 [待确认]）：
- `background`：背景（为什么现在做）
- `acceptance_criteria`：验收标准
- `out_of_scope`：不做什么（边界）
- `metrics`：成功指标

## 章节结构（固定骨架）

```
# {feature_name} PRD

## 一、背景与目标
### 背景
{background 或 [待确认]}

### 目标
{goal_one_liner}

### 不做什么
{out_of_scope 或 [待确认]}

## 二、目标用户
{target_user}

## 三、功能需求
（按用户故事组织：作为<角色>，我希望<动作>，以便<价值>）

## 四、验收标准
{acceptance_criteria 或 [待确认草稿]}

## 五、成功指标
{metrics 或 [待确认]}

## 六、开放问题
（生成时留空，后续补充）
```

## 风格示例

> # 消息已读回执 PRD
>
> ## 一、背景与目标
> ### 背景
> 当前群聊无法知道对方是否看到消息，导致重复追问。
>
> ### 目标
> 让发送方知道消息被接收，减少无效追问。
>
> ### 不做什么
> 不做单聊已读（本期只覆盖群聊）。

规范措辞、用户故事格式、验收标准可量化。
