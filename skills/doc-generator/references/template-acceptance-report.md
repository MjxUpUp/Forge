# 验收报告模板

## 变量列表

**必填**：
- `project_name`：项目名称
- `scope`：验收范围（哪些功能/模块）

**选填**：
- `prd_ref`：对应 PRD/需求文档链接
- `test_report`：测试报告摘要

## 章节结构（5 维度，参考 project-acceptance skill）

```
# {project_name} 验收报告

## 一、PRD 功能覆盖度
{逐条核对 PRD 需求 vs 实际实现，覆盖度 %}

## 二、设计方案 vs 实现一致性
{设计文档承诺 vs 代码实际，不一致项}

## 三、代码质量
{命名/错误处理/架构合理性/注释}

## 四、测试覆盖
{单元 + 集成 + E2E 覆盖情况，通过率}

## 五、文档与易用性
{README/API 文档是否更新，新手能否上手}

## 验收结论
{通过 / 有条件通过（列出待修项）/ 不通过}
```

## 风格示例

规范措辞、每维度有明确结论（通过/不通过+证据）、覆盖度用百分比。
